# v0.3 复核清单：素材库可用性增强

本清单用于 v0.3 开发和独立复核。v0.3 的目标是把“能上传素材”推进到“像真正客户端素材库一样可管理、可搜索、可选择”。

## 1. 安全必查

### 素材删除

- 删除接口只允许操作固定素材分类：`covers`、`backgrounds`、`introVideos`、`audio`。
- 删除目标必须解析到对应分类目录内：
  - `public/covers/`
  - `public/backgrounds/`
  - `public/intro-videos/`
  - `public/audio/`
- 不允许删除 `public/` 根目录里的示例音频等内置素材。
- 请求里的路径必须拒绝 `..`、绝对路径、反斜杠路径、远程 URL、`data:`、`blob:`。
- 最终路径必须通过 `withinBase(base, target)` 校验。
- 删除不存在文件应返回受控 404，不抛出未处理异常。

### 素材重命名

- 重命名接口只允许操作固定素材分类目录内的文件。
- 新文件名必须经过 `safeSlug` 清理。
- 扩展名必须保持原扩展名，或落在该分类白名单内；不允许改成 `.html`、`.svg`、`.js` 等可执行或可注入格式。
- 如果目标文件已存在，必须自动追加后缀或返回受控冲突，不得覆盖。
- 重命名后 `/api/assets` 应返回新路径，不再返回旧路径。

### 已有加固不得回退

- Host 防 DNS rebinding 仍在。
- 413 路径仍设置 `Connection: close`。
- 路由里的异步 handler 继续 `await`。
- 上传继续使用 MIME 白名单、扩展名白名单、`safeSlug` 和固定目录。
- 批量 `id` 清理、`books` 非对象降级、413 稳定返回、队列重试索引回写继续受回归测试保护。

## 2. Electron / Web 边界

- “打开所在目录”“系统播放器打开视频”属于 Electron 能力。
- Web 模式必须优雅降级为：
  - 复制路径；
  - 新标签打开可访问的 `/out/...` 或 `/public/...` 链接；
  - 文案提示需要桌面客户端才可调用系统文件管理器。
- 若加入 Electron IPC：
  - Renderer 不启用 Node integration；
  - 通过 preload 暴露最小 API；
  - 主进程侧仍做路径边界校验；
  - 不接受任意 URL 或任意本地路径直接打开。

## 3. 产品验收

- 素材库支持分类筛选。
- 素材库支持搜索。
- 素材卡片可预览缩略图或媒体类型。
- 素材卡片支持复制路径。
- 可上传目录内的素材支持重命名与删除，并能刷新列表。
- 主编辑页选择素材不再只靠长下拉，应提供可视化选择面板。
- 快闪书单逐本封面能从可视化面板选择。
- 任务中心支持播放 / 复制路径 / 打开链接的 Web 降级。
- 设置页展示当前素材目录和输出目录，暂不暴露任意目录托管，避免扩大文件暴露面。

## 4. 回归验证

- `node --check scripts/server.mjs`
- `node --check scripts/test-regressions.mjs`
- `npx.cmd tsc --noEmit`
- `npm.cmd run gui:build`
- `npm.cmd test`
- 带浏览器时应继续达到完整回归全绿。
- 浏览器实测：
  - 素材库搜索 / 筛选；
  - 素材重命名 / 删除；
  - 主编辑页可视化选择素材；
  - 设置页目录信息；
  - 任务中心 Web 降级操作。

## 5. 仓库卫生

- 不提交 `release/`。
- 不提交 `gui/dist/`。
- 不提交 `node_modules/`。
- 不提交 `out/`。
- 不提交用户上传素材和大二进制。
- 阶段完成后更新阶段文档，并把代码提交上传到 GitHub PR 分支。
