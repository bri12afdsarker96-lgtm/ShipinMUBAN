# 全项目复盘与阶段目标规划

更新时间：2026-07-19

## 1. 项目定位

ShipinMUBAN 的目标不是做一个单条视频样片，而是把“书籍分享短视频开头动画”沉淀为可配置、可替换、可批量生成的本地视频模板工具。

核心用户不是开发者，而是需要持续生产书籍类短视频的人。最终形态应当是：用户输入书单、主书、文案、音乐和素材，软件自动生成可发布的竖屏开场视频。

## 3. 产品主线

产品主线分为三条：

1. 视觉主线：复刻书籍分享开头的节奏、封面快闪、主书氛围页。
2. 数据主线：从固定样片升级为书单、主书、字幕、开场、音乐、素材全配置。
3. 生产主线：从单条渲染升级为批量导入、自动卡点、队列渲染、质检和归档。

这三条主线都已经有第一版实现，但第三条还需要真实使用场景验收。

## 5. 阶段 0：样片拆解与产品定义

### 目标

确认原视频开头动画的真正机制，并判断该机制能否软件化。

### 已确认结论

- 书籍介绍段不是慢速横向轮播，而是书封快闪硬切。
- 每张书封大约保留 4 到 8 帧。
- 快闪切点和音频瞬态有关，多数切点与音频瞬态差距不超过约 0.10 秒。
- 最小可复刻单元是“情绪钩子 -> 书封快闪 -> 主书氛围页”。

### 验收标准

- 有逐帧切点记录。
- 有默认切点帧列表。
- 能解释画面和音频如何卡点。

### 当前状态

已完成。相关记录在 `book-intro-maker/docs/analysis.md`。

## 6. 阶段 1：Remotion MVP

### 目标

用 Remotion 做出第一版可运行样片，验证技术路线。

### 已完成功能

- 竖屏 `720x1280 / 30fps`。
- 前几秒情绪钩子场景。
- 约 `4.4s-6.5s` 的书封快闪段。
- 最终进入主书氛围页。
- 示例节拍音频用于验证卡点。

### 验收标准

- `npm run render` 可以导出完整 MP4。
- 卡点段符合样片节奏。
- 没有真实封面时也能用生成式封面完成渲染。

### 当前状态

已完成。它证明 Remotion 是合适的渲染底座。

## 7. 阶段 2：配置化与真实素材

### 目标

把固定示例模板升级为可配置的书籍开场生成器。

### 阶段范围

- 书单配置化。
- 主书配置化。
- 独立字幕轨道。
- 前几秒本地视频可替换。
- 真实封面查询与缓存。
- 缺封面、缺网络、缺素材时自动降级。

### 已完成功能

- `config/books.example.json` 控制快闪书单、切点、主书。
- `config/subtitles.example.json` 控制独立字幕轨道。
- `config/intro.example.json` 控制开场视频或生成式背景。
- `scripts/fetch-covers.mjs` 支持 Open Library / Google Books 查询与本地缓存。
- `CoverImage` 加载失败后自动降级为生成式封面。

### 验收标准

- 修改配置即可替换书单和主书。
- 本地视频路径、裁剪、音量、静音可配置。
- 字幕不再写死在场景里。
- 无网络时仍可渲染。
- 网络正常时能下载并缓存真实封面。

### 当前状态

功能完成，但真实封面下载在当前本机网络下仍未闭环。

已执行 smoke test：`npm.cmd run covers`。结果是 Open Library / Google Books 均 `fetch failed`，4 本书全部降级为 `placeholder`。这说明降级链路可用，但真实封面链路需要在网络稳定环境或代理配置后复测。

## 8. 阶段 3A：批量生产引擎

### 目标

实现“一行数据生成一条视频”，为后续可视化软件提供后端能力。

### 已完成功能

- CSV / JSON 导入。
- 扁平字段和结构化三件套两种输入。
- 默认切点自动生成。
- 批量队列、并发上限、失败重试。
- 渲染前质检和渲染后文件校验。
- 输出 `manifest.json` 与 `qc-report.json`。

### 验收标准

- 至少 3 条不同主题数据 dry-run 通过。
- 单条异常不影响整批归档。
- 输出路径不能越界。
- 同名 id 不能覆盖。

### 当前状态

已完成，并已通过回归测试守护。

## 9. 阶段 3B：模板库、素材库、自动卡点

### 目标

让同一套视频结构可以快速换风格、换音乐、换素材，并更贴合音频节奏。

### 已完成功能

- 四套模板：`classic`、`healing`、`quote`、`drama`。
- 素材库：audio、covers、backgrounds、introVideos、subtitleStyles。
- 背景音乐可配置，不再硬编码。
- WAV 解码和能量新颖度瞬态检测。
- `beatsAudio` 可在批量渲染前自动生成 `flashCutFrames`。

### 验收标准

- 不同模板在主书页有明确视觉差异。
- 音频检测切点和人工标注误差约在 1 帧量级。
- 找不到素材引用时降级不中断。

### 当前状态

已完成。后续需要补的是对不同音乐类型的稳定性评估，尤其是氛围乐、无明显鼓点音乐。

## 10. 阶段 3C：可视化编辑器与本地服务

### 目标

让非开发用户不改代码也能编辑、预览、导出和渲染。

### 已完成功能

- React 可视化编辑器。
- `@remotion/player` 实时预览。
- 模板、音乐、主书、书单、字幕表单。
- 导出配置 JSON。
- 本地服务版支持 `/api/render` 一键出 MP4。
- 批量队列面板支持提交多条、轮询进度、查看产物。
- 批量历史可从磁盘归档恢复。

### 验收标准

- GUI 能打包。
- 本地服务能启动。
- 单条渲染接口返回可播放 MP4。
- 批量队列能显示状态和产物链接。

### 当前状态

Beta 完成。它是可用编辑器，不是成熟桌面产品。

下一步要补：文件选择、素材上传、封面查询按钮、自动卡点按钮、队列暂停/恢复/单条重试。

## 11. 阶段 3D：稳定性加固与回归测试

### 目标

把反查暴露的硬伤修掉，并用测试防止回归。

### 已修复硬伤

- 服务 async handler 未 `await` 导致 unhandled rejection 崩服务。
- `books` 非对象导致渲染目标关闭并打崩服务。
- 批量 `id` 可路径越界。
- 批量同名 `id` 会覆盖产物。
- 大请求体 413 返回不稳定，且 keep-alive 连接会变脏。
- 并发 worker 异常可冒泡导致整批失败。
- 0 字节输出未被识别为失败。
- CSV BOM 和 TSV 解析问题。
- 调试时间码默认烧进成片。

### 已建立保护

- `scripts/test-regressions.mjs` 覆盖三类合并前硬伤。
- `npm test` 在带浏览器环境下 11/11 通过。
- CI workflow 配置了 `RUN_RENDER_TESTS=1`，避免渲染项被跳过后假绿。

### 当前阻断

GitHub Actions 被账号 billing / spending limit 阻止启动。job 没有跑到 `npm ci`，不是代码或 workflow 已知错误。

### 当前状态

代码侧完成，真实 runner 验证等待账号侧恢复。

## 13. 阶段 4：产品化准备

### 目标

把开发者可用的 Beta 工具，推进成非技术用户可持续使用的软件。

### 任务规划

| 模块 | 目标 | 验收 |
| --- | --- | --- |
| 素材管理 | 在界面选择封面、音乐、背景、开场视频 | 用户无需手写路径 |
| 封面查询 | 界面内一键查询并缓存封面 | 可看到成功/失败/降级状态 |
| 自动卡点 | 界面内选择音频并生成切点 | 切点可预览和微调 |
| 队列控制 | 暂停、恢复、单条重试、失败原因 | 批量任务可被用户管理 |
| 视觉验收 | 自动导出关键帧截图 | 人工能快速检查画面 |
| 配置读写 | 导入/保存项目配置 | 用户可复用一套模板工程 |

### 不应优先做

- 复杂账号系统。
- 云端协作。
- 大规模模板市场。
- 过早桌面打包。

阶段 4 的原则是先把本地软件体验打磨顺，不急着扩成平台。

## 16. 阶段提交规则

每完成一个明确阶段，都要提交并上传到 GitHub 当前工作分支。

当前规则：

- 工作分支：`claude/shipinmubam-handoff-avn529`
- PR：#2
- 提交前检查 `git status`。
- 生成物不提交：`out/`、`public/covers/`、`config/*.resolved.json`、`gui/dist/`。
